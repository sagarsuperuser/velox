# Code of Conduct

The Velox project follows the [Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) without modification.

The full text — including the standards we hold ourselves to, the enforcement guidelines, and the four-tier consequence ladder (correction → warning → temporary ban → permanent ban) — is available at the link above. We adopt it verbatim rather than vendoring a copy that could drift.

## Reporting

To report a Code of Conduct concern, email **conduct@velox.dev**. Reports are read by the project maintainer only and are not made public.

If your concern overlaps with a security vulnerability, see [`SECURITY.md`](SECURITY.md) for the disclosure policy — we'll route appropriately.

## Scope

This Code applies to all project spaces — the GitHub repository (issues, pull requests, discussions, code reviews), any official Velox community channels (chat, forum, mailing list once they exist), and any in-person or virtual event held under the Velox name.

It also applies when an individual is officially representing the project — for example, using the `@velox.dev` email, posting from a Velox social account, or speaking at an event as a Velox maintainer.

## Enforcement

The maintainer at conduct@velox.dev is responsible for:

- Acknowledging every report within 5 business days
- Investigating in good faith with the reporter's stated comfort level on confidentiality
- Applying the Contributor Covenant enforcement ladder consistently
- Documenting the outcome in a private incident log retained for 18 months (matches the audit log retention default in `docs/ops/audit-log-retention.md`)

The maintainer is held to the same standard as everyone else; concerns about the maintainer's own conduct should be sent to **conduct-escalation@velox.dev** (a separate inbox).
