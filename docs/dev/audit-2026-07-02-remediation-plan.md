## 3. Fix-Design Notes — the three DP-blockers

These fold in every blocker/concern the design lenses raised. They are binding.

### 4.1 P1 — Threshold hardening

**Intra-PR order (each its own commit + mutation-verified test): (1) boundary skip → (2) fire-once probe → (3) cursor drain → (4) reset=true proration → (5) max/last → (6) $0 gate.** The drain MUST NOT precede the probe — without the probe, draining amplifies the per-tick tax-calc/invoice-number burn from 50 subs to the whole crossed set.

1. **Boundary = SKIP, not clamp.** In `scanOneThreshold`: `if !now.Before(*sub.CurrentBillingPeriodEnd) { return false, nil }`. The same-tick cycle scan (scheduler step 0.5 precedes step 1 in one goroutine; `scheduler.go:361→402`) bills the whole period through the full-fidelity path. Clamp-and-fire is money-correct (watermark protocol at `engine.go:2295`/`2402` closes both orders — verified, no gap, no double-bill) but bills the full period through fireThreshold's crude path: unprorated in_arrears base, no item-change segments (`engine.go:2158`/`2299`), no terminal-cancel in_advance skip (`engine.go:2121`), no in_advance header shift — mis-billing exactly the subs with a mid-period swap or scheduled cancel. Accepted semantic loss: no `subscription.threshold_crossed` event for a crossing first observed on/after the boundary tick — assert in test, note in ADR/CHANGELOG.
2. **Fire-once probe** placed in `scanOneThreshold` **BEFORE `evaluateThresholds`**: `LatestThresholdPeriodEnd(ctx, sub.TenantID, sub.ID, periodStart, periodEnd)`; non-ErrNotFound hit → `return false, nil`. This also skips the full previewWithWindow aggregation for fired subs — what makes the drained scan affordable. Preserves today's reset=true-advance-failed repair path (cycle-close watermark, `engine.go:2123-2142`, never a fire retry). **The probe is check-then-act, NOT the exactly-once mechanism** — the unique index is the correctness seam (document in the probe's comment; never write "CAS already guarantees exactly-once" in the PR).
3. **Cursor drain:** add `afterID` to `ListWithThresholds`/`ForClock` (`WHERE s.id > $cursor ORDER BY s.id LIMIT batch`); loop in `ScanThresholds` until short batch; cursor advances past a failing sub (never re-process within one tick). No max-pages guard (strictly-increasing cursor can't loop; no belt-and-suspenders). Verify EXPLAIN on the cursored query once in the integration test (the 0056 index may not serve it). ## 5. Rules of Engagement

1. **Panels are lean and pipelined.** The audit already did the adversarial discovery; a panel's job now is fix-shape review — one adversarial pass against the enumerated site-set, no multi-agent fan-out. Panels required: P1 (fix 5 only), P2b (×2), P14, P4, P3 (short), P5 (short — lease arithmetic + crash points). P8 gets site-set enumeration only. End each window by running the next day's panel on residual tokens so implementation starts at window-open.
2. **Site-set enumeration in every PR body** (playbook rule: never reason locally). The enumerations in §4 are starting points, not substitutes.
3. **Self-merge gate:** green CI + mutation-verified regression test + docs riders (CHANGELOG, MANUAL_TEST flow, ADR where a decision was made) in the same PR. Verify the real path, not a shortcut; positive AND negative assertions.
4. **Worktree discipline:** one worktree per PR off origin/main; never open a lane-successor's worktree until the predecessor is merged (a conflicting PR gets no CI — every out-of-order push burns a ~6-min cycle + rebase). Rebase onto origin/main before push, always.
5. **Quota pacing:** money-PR-first each window; S riders fill residue; if a window opens with the previous day's lane PR unmerged, finish it before starting anything new. CI is the only integration gate (shared local Postgres — no parallel `-short=false` runs).
6. **Migration/ADR numbers** claimed Day 1 across origin/main + all local branches; drop-table downs recreate (feature_flags down re-seeds 0001/0002 rows).
7. **Tracking:** the deduped report (16/51/50) is the ledger; each merge marks its findings AND its dupe-roots closed. Lows stay deferred except the 9 named ride-alongs — no opportunistic low-fixing outside owned files.
8. **Decisions locked in this doc** (archive=409-block; feature-flags=delete; boundary=skip; STARTTLS=strict; max/last-iff-reset-false; duplicate-PI-only escalation) are not reopened mid-window; deviating requires amending this doc first (amend-decisions rule).

## 6. Risks

1. **Quota exhaustion (proven: Jul 2's window died mid-audit).** A panel-heavy money day can consume a full window. Mitigation: lean panels, pipelining next-day panels into residual tokens, and the tiered cut-line so a lost window drops Tier 3 — never Tier 1.
2. **P1 blowup.** Six fixes + engine-parity + real-Postgres mutation tests in the product's weakest surface, heading the critical path. Mitigation: the P1a/P1b split — the double-billing kernel (skip + probe + drain) merges Day 1 regardless; P1b's panel-gated fix 5 can slip a day without moving P2b (which touches a different engine function and rebases trivially).
3. **Serialization churn on the billing lane.** Three engine.go successors + CI-as-only-integration-gate. Mitigation: strict merge order (§2 invariant), disjoint-function edits (rebases are mechanical), and parallel lanes (P5/P6/P9/P10/P13/P14) that provably never touch lane-A files.
4. **P4 decision stall.** The only unmade product decision in a money batch (retroactive vs pinned repricing) sits behind three merges. Mitigation: the decision + ADR happen Day 1 in design-space only; P4-impl starts Day 3 against a settled ADR; pin-at-consumption implementation is pre-authorized to spill. P10's `:401` slice waits for the same ADR — cross-batch regression risk eliminated by construction.
5. **Weekend capacity.** Jul 4–5 carry P1b, P2b, P4, P5-core, P6, P9-core. If a weekend window is lost: execute Tier 1 only; P4 and P7-core move to Day 4; everything else spills per the cut-line — no renegotiation.
6. **P2b hidden-writer regression.** The engine trial branch (`engine.go:1941`) is the fix the original plan missed; skipping it re-ships the audit bug on a timer. Mitigation: test (11) in P2b's matrix drives activation through the engine cycle path specifically, clock-pinned.
7. **P7 verification is wall-clock, not CI.** Docs/compose have no mutation-test surface. Mitigation: P7-core scoped to the three code items with real tests (bootstrap, APP_DATABASE_URL fail-closed, migrate-lock); the walkthrough is the doc riders' gate, and those riders are pre-authorized spill; scheduled Day 4 so a spill lands in Day 5 buffer, not past the deadline.
